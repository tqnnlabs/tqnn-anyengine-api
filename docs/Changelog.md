# Versioning Policy

TQNN follows semantic versioning whenever practical.

- **Major** versions introduce breaking API changes.
- **Minor** versions introduce new functionality while maintaining compatibility.
- **Patch** versions include bug fixes, documentation updates, and performance improvements.

---

## Support Policy

The latest major API version receives active development and documentation updates.

Older versions may continue operating for a transition period but may not receive new features.
